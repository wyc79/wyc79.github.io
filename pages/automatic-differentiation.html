Why

Derivatives show up everywhere in optimization, ML, and scientific computing, but the common alternatives are frustrating in practice: numerical differentiation can be unstable (step-size tradeoffs), and symbolic differentiation can blow up in expression size.

We wanted a small, readable tool that computes exact derivatives at runtime, so users can differentiate ‚Äúnormal Python code‚Äù without manually applying the chain rule.

What

A forward-mode automatic differentiation (AD) Python package built by a 5-person team. It evaluates user-defined functions and returns derivatives by propagating dual numbers through operator overloads and elementary functions.

The focus was a clean API, strong correctness guarantees, and maintainability through a test-first workflow.

How

My contributions (Core AD + QA)

I implemented key pieces of the forward-mode engine and led QA to keep the package correct as features expanded.

  • Forward-mode engine: Implemented dual number arithmetic and operator overloading to propagate derivatives through composed expressions.
  • Function composition: Ensured derivatives flow correctly through chained operations (add/mul/div/pow, etc.) by applying the chain rule consistently across overloaded operators.
  • Refactoring for clarity: Reduced redundancy and improved readability so teammates could extend the system without breaking core invariants.
  • QA leadership: Designed and executed comprehensive unit tests (core ops + edge cases) to verify numerical correctness and prevent regressions.
  • Debugging & integration: Coordinated module integration, triaged bugs, and documented design decisions to streamline collaboration.
  • Release readiness: Helped ensure stable packaging/behavior for publishing on TestPyPI.
