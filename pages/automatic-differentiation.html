Why为什么做

Derivatives show up everywhere in optimization, ML, and scientific computing, but the common alternatives are frustrating in practice: numerical differentiation can be unstable (step-size tradeoffs), and symbolic differentiation can blow up in expression size.导数在优化、机器学习和科学计算中无处不在，但常见的几种做法在实践中都很别扭：数值微分不够稳定（受步长权衡的影响），符号微分则可能让表达式规模急剧膨胀。

We wanted a small, readable tool that computes exact derivatives at runtime, so users can differentiate “normal Python code” without manually applying the chain rule.我们想要一个小巧、易读的工具，能在运行时计算精确的导数，让用户可以对“普通的 Python 代码”求导，而无需手动套用链式法则。

What做了什么

A forward-mode automatic differentiation (AD) Python package built by a 5-person team. It evaluates user-defined functions and returns derivatives by propagating dual numbers through operator overloads and elementary functions.一个由 5 人团队开发的前向模式自动微分（AD）Python 包。它对用户自定义的函数求值，并通过在运算符重载和基本函数中传播对偶数来返回导数。

The focus was a clean API, strong correctness guarantees, and maintainability through a test-first workflow.开发重点在于简洁的 API、可靠的正确性保证，以及通过测试先行的工作流带来的可维护性。

How如何实现

My contributions (Core AD + QA)我的贡献（核心 AD + QA）

I implemented key pieces of the forward-mode engine and led QA to keep the package correct as features expanded.我实现了前向模式引擎的关键部分，并主导 QA 工作，在功能不断扩展的同时确保包的正确性。

  • Forward-mode engine: Implemented dual number arithmetic and operator overloading to propagate derivatives through composed expressions.
  • 前向模式引擎：实现了对偶数运算和运算符重载，以在复合表达式中传播导数。
  • Function composition: Ensured derivatives flow correctly through chained operations (add/mul/div/pow, etc.) by applying the chain rule consistently across overloaded operators.
  • 函数复合：通过在各重载运算符中一致地应用链式法则，确保导数在链式运算（加/乘/除/幂等）中正确传递。
  • Refactoring for clarity: Reduced redundancy and improved readability so teammates could extend the system without breaking core invariants.
  • 重构以提升清晰度：减少冗余、改善可读性，让队友能在不破坏核心不变量的前提下扩展系统。
  • QA leadership: Designed and executed comprehensive unit tests (core ops + edge cases) to verify numerical correctness and prevent regressions.
  • QA 主导：设计并执行了全面的单元测试（核心运算 + 边界情况），以验证数值正确性并防止回归。
  • Debugging & integration: Coordinated module integration, triaged bugs, and documented design decisions to streamline collaboration.
  • 调试与集成：协调模块集成、分类处理缺陷，并记录设计决策，以让协作更顺畅。
  • Release readiness: Helped ensure stable packaging/behavior for publishing on TestPyPI.
  • 发布准备：协助确保打包与行为的稳定，以便在 TestPyPI 上发布。
