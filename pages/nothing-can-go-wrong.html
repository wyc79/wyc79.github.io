Nothing Can Go Wrong

Role: Primary Programmer
Tools: Godot 4, GDScript

Play on itch.io

What

Nothing Can Go Wrong is a fast-paced game created during a game jam, where the goal was to design and ship a complete playable experience within a very short development cycle.

I served as the primary programmer, responsible for implementing the core gameplay systems, player controls, and game logic while integrating assets created by teammates.

The project focused on rapid iteration and experimentation while maintaining a stable, playable build throughout the jam.

Main Achievements

  • Implemented the core gameplay systems and player control logic.
  • Integrated game mechanics, UI feedback, and input systems using Godot.
  • Rapidly iterated on gameplay mechanics under strict time constraints.
  • Delivered a polished playable build within the game jam deadline.

Highlights

How

Rapid Gameplay Implementation

Because of the short jam timeline, the main engineering goal was to build gameplay systems that were simple, modular, and easy to iterate on.

  • Implemented core gameplay mechanics and player control logic using Godot and GDScript.
  • Built systems that connected input handling, game state, and player feedback.
  • Structured scripts so new mechanics could be quickly added or adjusted during playtesting.

Rapid Iteration and Debugging

Game jams require constant iteration and quick problem solving. The development process involved frequent playtesting and adjustment to ensure the game remained fun and stable.

  • Iterated on gameplay mechanics based on quick playtest feedback.
  • Debugged gameplay issues and optimized scripts to maintain smooth performance.
  • Ensured the game remained stable while new features were added rapidly.
